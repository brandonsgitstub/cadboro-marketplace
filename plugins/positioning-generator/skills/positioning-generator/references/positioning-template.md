# Positioning Brief — Output Template

The structure for the positioning brief. The default deliverable is a `.docx` (use the docx skill). Use `.md` only if the user asks. Match section depth to how much was actually decided. Never pad.

Order the brief so a reader who wasn't in the room gets the answer first, then the reasoning.

---

## Section structure

### Title block
- Product name
- "Positioning Brief"
- Status: Draft
- Date
- One line on what this covers (new positioning vs repositioning)

### 1. The positioning statement
Lead with it. This is the artifact people will quote. Provide two forms:

**Classic statement** (the canonical, fill-in form):
> For [target customer] who [need or trigger], [product] is a [market category] that [key value]. Unlike [primary competitive alternative], [product] [primary differentiator].

**Short narrative** (2–3 sentences, for internal alignment and the top of a deck). Plain, declarative, no hype.

### 2. The five components
A clean table or section per component, filled from the work:

| Component | The answer |
|-----------|-----------|
| Competitive alternatives | What they'd use instead, including DIY / do-nothing |
| Unique attributes | Concrete, checkable, true vs the alternatives |
| Value (and proof) | Benefit each attribute enables + proof or proof-path |
| Target market | The beachhead segment, defined by identifiable characteristics |
| Market category | The frame, plus which of the three category strategies and why |

If a relevant trend was used, add it as a sixth row with a note on why it's safe to lean on.

### 3. Category rationale
One short paragraph. Why this category frame, what it signals about competitors and price, and what it costs to claim. If recommending a new category, state the cost and risk plainly and say what it would take to pull off.

### 4. Why this beachhead
One short paragraph. Why this segment over the others, who the secondary segments are, and the signal a marketer would use to find the primary segment.

### 5. What this unlocks
A few bullets teeing up downstream work without doing it: the messaging angles this opens, the competitive contrasts to lean on, the proof points to go collect. This is the handoff to the Messaging Framework skill. Keep it to hooks, not copy.

### 6. Assumptions to verify
If any component was assumed rather than known, list every `[ASSUMED]` item here in one place so the user can confirm before sharing. If everything was grounded in user input, say so and drop this section.

---

## Voice for the brief

The brief itself follows house voice: punchy, dry, short sentences, no em dashes. Declarative over decorated. The positioning statement especially must read like a person wrote it. If a line sounds like a tagline a committee approved, rewrite it.

Banned in the output: "seamless", "robust", "intuitive", "game-changing", "revolutionary", "best-in-class", "cutting-edge", "leverage" (as a verb), "synergy". If one of these is the only word that fits, the thought underneath it is too vague. Sharpen the thought.
