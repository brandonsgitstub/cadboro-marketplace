# Positioning Frame & Method

The reference for *how* to build positioning. Read this before working through the components. The output structure lives in `positioning-template.md`.

This skill uses a Dunford-style frame (from April Dunford's *Obviously Awesome*), adapted for fast SaaS work. The goal is not academic completeness. The goal is a positioning statement a founder can defend and a team can rally around by end of day.

---

## The core idea

Positioning is context-setting. You are choosing the frame of reference a buyer uses to understand what you are, so that your strengths look obvious and your weaknesses look irrelevant. Get the frame wrong and even a great product reads as "meh, like everything else." Get it right and the same product reads as "obviously the one I want."

Positioning is a deliberate choice, not a tagline and not a mission statement. It is upstream of messaging. This skill produces the choice. The Messaging Framework skill produces the words.

## The five components (plus one)

Work them in this order. Each one constrains the next.

### 1. Competitive alternatives
What would the customer use if you did not exist? This is the anchor for everything, and it is almost never "our named competitor." It is usually:
- A direct competitor (sometimes)
- A homegrown / DIY solution (spreadsheets, scripts, a Notion doc)
- A manual process (hire someone, do it by hand)
- Do nothing / live with the problem

Push hard here. The real alternative is what sets the buyer's expectations. If the honest answer is "they use a spreadsheet," your positioning is against the spreadsheet, not against the VC-funded competitor nobody has heard of.

### 2. Unique attributes
What do you have that the alternatives do not? Features, capabilities, model, data, integrations, business model. Be concrete and be honest. Only list what is actually true and actually distinct against the alternatives in component 1. "Easy to use" is not an attribute. "One-click import from QuickBooks" is.

### 3. Value (and proof)
For each unique attribute, what value does it enable that the customer cares about? Map attribute to benefit. Then ask: can we prove it? Value with no proof is a claim. Value with proof is positioning.

Attribute → Value example:
- Attribute: "Imports directly from the customer's CRM"
- Value: "Reps see deal context without leaving the tool, so they prep in minutes not hours"
- Proof: "Customers report cutting call prep from 30 min to under 5"

### 4. Target market characteristics
Who cares *a lot* about that value? Not your TAM. The customers for whom the value is most acute and most urgent. Define them by identifiable characteristics, so a marketer could actually find them:
- Firmographics (size, stage, industry, model)
- Situational triggers (just raised, just scaled the team, just got burned)
- Observable signals (uses tool X, has role Y, structured a certain way)

The best-fit segment is the one where your unique value is a need-to-have, not a nice-to-have.

### 5. Market category
What context do you put yourself in so the value is immediately obvious? The category is the buyer's mental shelf. It sets their expectations for what you do, who you compete with, and what you should cost.

Three category strategies:
- **Play in an existing category** you can win a sub-segment of. Lowest cost. Default choice.
- **Position against a category** ("the anti-CRM"). Useful when an incumbent category has trained buyers into a pain you solve.
- **Create a new category.** Highest cost, highest risk, rarely correct. Only when no existing frame makes your value make sense, and you have the resources to teach the market. Flag this loudly if you recommend it.

### 6. Relevant trends (optional, use with care)
A trend that makes you newly relevant ("now that every team has remote workers..."). Layer this on only if it is genuinely true and genuinely helps. Trend-stacking is a common way to sound like vaporware. If in doubt, leave it out.

## The process

1. **Establish what we're positioning.** New product, or repositioning an existing one? If repositioning, capture the current (probably broken) positioning and *why* it's failing before proposing the new one.
2. **List the true competitive alternatives.** Force the DIY / do-nothing options onto the list.
3. **Isolate unique attributes** against those alternatives.
4. **Map attributes to value, and value to proof.** Drop attributes that map to no value the buyer cares about.
5. **Find the segment that cares most.** If several, pick a beachhead as primary and name the others as secondary. Do not position for everyone; positioning for everyone is positioning for no one.
6. **Choose the category frame** using the three strategies above.
7. **Write the statement** (see template).
8. **Pressure-test it** with the checklist below.

---

## Pressure-test checklist

Before finalizing, run the positioning against these:
- Would the chosen segment agree the competitive alternative is what they'd actually use? If not, the anchor is wrong.
- Does every value claim have at least a path to proof? Strip or flag the ones that don't.
- Could a competitor say the exact same thing? If yes, it is not differentiation, it is table stakes.
- Does the category set the right price expectation and the right competitors?
- Is the target specific enough that a marketer could build a list? "SMBs" is not. "Series A SaaS companies with a 5 to 15 person sales team using HubSpot" is.
- Read the one-liner aloud. Does it sound like a human said it, or like a pitch deck generated it?

## Handling thin input

If the user gives you little to work with, you can still produce a useful draft. Make explicit, reasonable assumptions for the competitive alternatives, segment, and category, mark each one `[ASSUMED \u2013 confirm]`, and produce the brief. A flagged draft the user can react to beats an interview that stalls. But if the *core* facts are missing (you don't know what the product does or who it's roughly for), ask before drafting. You cannot position a black box.

## Common failure modes to avoid

- **Positioning against the wrong alternative** (a tiny named competitor instead of the spreadsheet everyone actually uses).
- **Listing benefits as attributes** ("powerful", "flexible"). Attributes are concrete and checkable.
- **Targeting the TAM instead of the best-fit beachhead.**
- **Reaching for a new category** when an existing frame would do.
- **Trend-stacking** to sound current.
- **Aspirational positioning** that describes the product you wish you had, not the one you can prove today.
